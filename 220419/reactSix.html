useState

2022-04-19

prop은 컴포넌트를 사용하는 외부자를 위한 데이터이고, state는 컴포넌트를 만드는 내부자를 위한 데이터이다.

본문을 클릭하면 그에 해당하는 상세페이지가 나오고, 홈을 클릭하면 홈의 내용이 나오게 만들고자 한다.

  1. App함수 내에 mode의 초기값을 지정한다
  2. 만약 mode의 값이 문자열 WELCOME이면 나타날 콘텐츠와, READ이면 나타날 콘텐츠를 작성한다.
    • content 변수를 선언하고 초기값을 지정하지 않은 null로 설정.
  3. content는 원래 Article 컴포넌트가 있던 자리에 중괄호로 작성한다.
  4. 해당 속성에 mode="문자열"; 작성
    • 이 상태로 Nav영역을 클릭해도 콘텐츠가 달라지지 않는다. App 함수는 다시 실행되지 않기 때문에 새로운 return 값을 반환할 수 없다.
  5. 그래서 필요한 것이 useState 명령어. 상단에
    import {useState} from 'react'
    를 작성한다. 이는 지역변수를 상태(state)로 업그레이드 하는 과정이다.
  6. useState는 배열로 return 된다.
    const _mode = useState('WELCOME');
    👉 _mode에는 [상태의 값, 상태의 값 변경 함수]가 들어가 있는 상태.
    const mode = _mode[0];
    👉 mode에는 배열에서 index 0의 자리에 있는 상태의 값만을 읽음.
    const setMode = >mode[1];
    👉 setMode에는 배열에서 index 1의 자리에 있는 함수를 가지며, 이를 통해 mode의 값을 바꿀 수 있다.
    • 위 세가지를 합쳐서 이런 식으로 작성한다.
  7. setMode를 이용하여 mode의 값을 바꿈
    • setMode로 인해 바뀌면 App 함수는 다시 실행되며 useState가 해당 값을 실행할 수 있게 된다.

nav의 해당 세부 콘텐츠가 보이게 하고 싶다면

  1. const [id, setId] = useState(null);
    을 선언한다.
  2. Nav의 이벤트 함수에 setId를 사용한다.
    • 매개변수는 상수 등 다른 id와 혼동이 있을 것 같아 _id로 변경하였다.
  3. 배열 topics에서 일치하는 id의 원소를 찾아 title과 body의 값으로 세팅
  4. content의 값을 작성한다.
  5. props의 값은 태그의 속성이 되면 문자열로 자동 변환되기 때문에 문자열이 된 id 값을 숫자로 변환해야 제대로 동작이 가능하다.
    • props 선언한 속성에 숫자로 컨버팅하는 함수 Number를 작성한다.
  6. 제대로 작동한다!
